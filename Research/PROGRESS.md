# Lw-Eh Progress Ledger

This is the durable, git-tracked ledger for the autonomous build loop. Updated at the end of every firing. Git history is ground truth if this ever drifts — see `git log --oneline`.

- **Phase:** core-implementation (delegate done, signal next)
- **Last completed:** Implemented `lweh::delegate<Sig>` (`include/lweh/delegate.hpp`) — the Ryazanov-style non-owning 2-pointer bind/invoke type (research.md §A1). Investigated first via 2 parallel subagents (toolchain/GCC-version portability of `template<auto Fn>` NTTP across ESP-IDF/arm-none-eabi-gcc/avr-gcc; EnTT cross-validation + language-correctness of virtual dispatch and inherited-method binding through a compile-time-captured pointer-to-member). Both confirmed the design is safe as-is; the one concrete implementation detail both surfaced — the free-function stub must use direct call syntax, not `std::invoke` (which would silently reinterpret a misused member pointer as an "unbound member" call) — is baked in, plus a defensive `static_assert(!std::is_member_pointer_v<decltype(Fn)>, ...)`. Added a permanent `static_assert(sizeof(delegate<void()>) == sizeof(void*)*2, ...)` regression guard directly in the header. Wrote `tests/delegate_test.cpp` (7 cases: free function, member function, cross-instance isolation, virtual dispatch through a base pointer, method inherited from a non-virtual base, default-unbound vs. bound truthiness, multi-arg/non-void signature) and registered it in `tests/CMakeLists.txt`. Judged as a "big" change (first real implementation, defines the delegate's binary layout) — full configure + full build + full `ctest` run performed, all 6 tests pass. Also fixed a real (if minor) papercut found along the way: `CMakeLists.txt` didn't export `compile_commands.json`, so clangd was falling back to an approximate `.clangd` config and misreporting "file not found" on every `<lweh/...>` include across the whole tree — added `CMAKE_EXPORT_COMPILE_COMMANDS ON` and pointed `.clangd` at `build/host-debug/compile_commands.json`.
- **Current measured size:** `sizeof(lweh::delegate<Sig>)` = 2 pointers, compiler-enforced on every build (16 bytes on this 64-bit host; will be 8 bytes on the 32-bit targets Lw-Eh actually ships to — ESP32/ARM/AVR all use 32-bit pointers). No meaningful non-zero *incremental footprint* number yet: a direct host-proxy probe (minimal `control.cpp` vs. `with_delegate.cpp`, both built under the real aggressive flag set — `-Oz -fno-exceptions -fno-rtti -flto=full -ffunction-sections -fdata-sections -Wl,--gc-sections -s`, etc.) produced **byte-identical** binaries even after forcing the input through a `volatile` to defeat constant-folding — under full LTO, a single statically-known delegate bind+invoke in one TU inlines away to nothing, empirically confirming the zero-overhead design but meaning delegate alone can't produce a non-zero size data point. A real number needs `signal<Event,N>`'s runtime-varying multi-listener dispatch loop (next up), since a signal iterating over N *possibly-different* bound callbacks is what the optimizer can no longer fully collapse. Separately noted: `size_audit/`'s current wiring (reusing `examples/esp32_minimal/main.cpp`, which links with `-T esp32.ld -nostdlib`) doesn't produce a valid link on a native host target, so the `size-host` CMake preset from Phase 0 doesn't actually work yet as configured — it needs its own host-appropriate control/with-lweh source pair rather than reusing the ESP32-specific example. Tracked as a follow-up, not fixed this firing (out of scope for "implement delegate").
